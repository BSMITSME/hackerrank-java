문제 설명
랭킹 시스템
- 가장 점수가 높은 사람의 랭킹은 1
- 만약 같은 점수라면 같은 랭킹으로 배정
- 정답값은 player의 각 랭킹을 return (랭킹 산정 즉시 정답 array에 반영)

첫번째 시도방법
1. ranked 배열에 player 값을 하나씩 넣어주며 정렬 및 Map에 작성
2. ranked와 Map도 값이 들어올때마다 새로 갱신 -> 답을 도출하는 과정이 값이 들어올 당시 순위를 산출
3. timelimit 문제 발생

시간복잡도 문제
순서 보장, 인덱싱(위치 접근 가능) -> 트리 알고리즘? heap 정렬 고려 해보기  -> 별로

두번째 시도방법 (복사)
- 이진탐색
첫번째 접근방법은 하나하나 확인 -> 순차탐색
이진탐색은 시작점, 끝점, 중간점을 이용하여 탐색 범위를 설정하는 방식 
모든 범위를 탐색하는 것이 아니라서 timelimit 문제 해결
